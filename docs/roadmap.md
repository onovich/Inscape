# 路线图

状态：基线 + 草案

## 阶段 1：奠基，2026 Q2

目标：证明从纯文本 DSL 到 Unity 可执行指令流的主链路。

计划内容：

- 完成基础 DSL 语法定义草案。
- 实现图叙事解析器原型，输出 AST、诊断和 Narrative Graph IR。
- 实现哈希锚点生成与增量本地化提取原型。
- 提供 VSCode 语法高亮、补全、诊断、跳转定义和大纲能力。
- 提供轻量 HTML 调试预览，支持节点跳转、选项、回环和路径记录。
- 开发 Unity 宿主解释执行器。
- 支持基础对话、背景切换、立绘切换、简单条件和分支。
- 调研 Bird 项目中 StorySystem/DirectorSystem 的映射边界，确认第一版是否只映射 Talking/L10N，还是包含 Timeline 引用。

阶段门槛：

- 一个短篇样例可以完整编译并在 Unity 中运行。
- 文本修改后能重新导出本地化表并标记差异。
- 存档能恢复到确定脚本位置和状态。
- VSCode 中可以获得基础高亮、补全和缺失节点诊断。
- HTML 预览可以在无 Unity 环境下调试叙事图。

## 阶段 2：流光，2026 Q3

目标：建立独立编辑器与实时预览闭环。

计划内容：

- 推出 Tauri 编辑器 Alpha。
- 集成 Monaco Editor 语法高亮、诊断和大纲视图。
- 实现编辑器与 Unity 的实时同步预览。
- 实现变量与状态监视器。
- 探索 Time Travel 调试。

阶段门槛：

- 作者可以在编辑器中完成脚本编写、诊断和预览。
- Unity 预览窗口可接收局部修改并刷新安全区域。
- 状态监视器能解释当前变量和执行位置。

## 阶段 3：共鸣，2026 Q4

目标：开放扩展能力，并优化多端运行体验。

计划内容：

- 插件化扩展机制。
- 自定义指令接入 C# 或脚本扩展。
- 移动端和 WebGL 的执行效率与资源加载优化。
- 完善开发者手册。
- 研究从 Ink 或 Yarn Spinner 迁移脚本的自动化工具。

阶段门槛：

- 第三方项目能注册并运行自定义指令。
- 编辑器能理解扩展指令的基础 Schema。
- 样例项目可以构建到移动端或 WebGL。

## 阶段 4：超脱，2027+

目标：探索脱离 Unity 的完整 Inscape 技术栈。

计划内容：

- 适配 Bevy 或自研轻量级渲染层。
- 抽象 Runtime Host 接口。
- 探索 AI 辅助创作流，包括分支建议、文本润色和翻译对齐。

阶段门槛：

- 同一份 IR 可被 Unity 外的运行时消费。
- 独立运行时完成基础视觉小说演出。
