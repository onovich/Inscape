# 待确认问题

状态：持续维护

这里记录当前最值得讨论的问题。解决后应更新对应文档，必要时新增 ADR。

## 最高优先级

1. DSL 的最小可用语法是什么？
   - 需要确认对白、旁白、标签、变量、条件、选择、跳转、注释和跨文件引用的表达方式。

2. 哈希锚点更偏向文本敏感，还是位置稳定？
   - 这会直接影响本地化复用、存档兼容和热重载定位。

3. 是否允许显式稳定 ID？
   - 隐式哈希很优雅，但关键节点、存档和本地化可能需要人工锁定能力。

4. 编辑器主交互是文本优先，还是文本与图双主视图？
   - 这会影响 Monaco 语言服务、逻辑图、热重载和用户心智模型。

5. Unity 目标版本到底是什么？
   - 当前材料同时出现 Unity 6 与 2023 LTS，需要统一表达和兼容策略。

## 语法设计

- 角色名和对白分隔符是否兼容中英文冒号。
- 旁白是否允许裸文本。
- 缩进是否有语义。
- 条件块是否使用 `end`。
- 选项语法如何兼顾阅读和结构化。
- 标签参数是否支持命名参数。
- 是否支持作者自定义标签。
- 是否需要宏或模板。
- 跨文件结构如何组织章节、场景和跳转。

## 编译器

- Antlr4 和 Superpower 谁更适合第一版。
- 错误恢复需要做到什么程度。
- IR 是否使用 JSON、MessagePack、ScriptableObject，还是多格式输出。
- 源映射格式如何设计。
- 编译器是否以库优先，CLI 作为薄封装。

## 编辑器

- 实时解析频率如何控制。
- 逻辑图是否可编辑。
- 预览端不在线时如何退化。
- 热重载 Patch 的最小协议是什么。
- 状态监视器是否允许修改变量。
- Time Travel 是否是阶段 2 必须功能，还是探索功能。

## 运行时

- Command Pipeline 是否足够，是否需要 ECS。
- 存档是否记录完整 Action 日志。
- 随机数和异步加载如何保证可回放。
- 自定义指令如何注册、验证和调试。
- Unity Addressables 是否作为第一版强依赖。

## 本地化

- CSV 和 PO 谁作为第一版默认格式。
- 重复文本是否共享译文。
- 文本轻微修改时如何继承旧译文。
- 文件改名和章节移动如何避免 Key 大量变化。
- 是否需要翻译上下文字段，例如角色、场景、前后句。
