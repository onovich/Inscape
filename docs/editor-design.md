# 编辑器设计草案

状态：基线 + 草案，交互未定稿

## 产品定位

Inscape 编辑器是叙事创作者和开发者共享的独立工作台。它不只是文本编辑器，也不是一开始就复杂化为完整节点图系统。第一阶段应该围绕脚本文本、诊断、大纲、预览和状态观察建立高效闭环。

正式独立编辑器属于第三步，但 DSL 阶段就需要提供轻量工具链：VSCode 语言支持和 HTML 调试预览。它们不是最终产品形态，而是帮助语言设计快速反馈的工作台。

## DSL 阶段 VSCode 支持

第一阶段需要尽量降低用户记忆负担，代码提示不是锦上添花，而是 DSL 可用性的核心部分。

目标能力：

- TextMate 语法高亮：弱化节点元信息、注释、跳转符号和标签，凸显剧情文本、角色名和选项文本。
- 语义高亮：区分节点定义、节点引用、角色、对白、旁白、选项、元信息和诊断区域。
- 自动补全：节点名、角色名、资源别名、选项模板、跳转目标和未来的查询谓词。
- 跳转定义：从 `-> node.name` 跳到节点定义。
- 引用查找：查看某个节点被哪些选项或跳转引用。
- 诊断：重复节点、缺失目标、不可达节点、疑似死循环、孤立节点、空选项、未闭合块、重复角色别名。
- 悬浮说明：解释节点、锚点、补全项和宿主 Schema。
- 大纲视图：按文件显示节点、选项和跳转。

## DSL 阶段 HTML 调试预览

第一阶段后期需要提供类似 Inky 的轻量 HTML 预览。

目标能力：

- 从编译产物读取 Narrative Graph IR。
- 显示当前节点的对白、旁白和选项。
- 点击选项进入目标节点，允许回环。
- 显示当前节点名、行级 hash、本地化 key 和诊断信息。
- 提供返回、重开、路径记录和简单搜索。
- 支持通过项目级 CLI `--entry node.name` 从任意节点临时启动预览。
- 不追求演出美观，不模拟 Unity Timeline，不加载复杂素材。

当前 CLI 已经支持单文件/项目级静态 HTML 预览。VSCode WebView 预览尚未接入，建议后续复用 CLI 生成的项目级 IR 与现有预览渲染逻辑。

## DSL 阶段本地化工具

当前 CLI 已经支持：

- `extract-l10n` / `extract-l10n-project`：导出当前可翻译文本。
- `update-l10n` / `update-l10n-project`：读取旧 CSV，按 `anchor` 精确继承译文，并标记 `current/new/removed`。

当前 VSCode 原型已提供命令面板入口，使作者不用离开编辑器即可导出或更新翻译表：

- `Inscape: Export Localization CSV`
- `Inscape: Update Localization CSV From Previous Table`

如果当前 `.inscape` 文件未保存，命令会通过 CLI `--override` 使用编辑器中的临时内容。

模糊匹配与人工确认 UI 后续再设计。

## 目标工作流

1. 作者打开项目和脚本文件。
2. 第一阶段在 VSCode 中编写 DSL，第三阶段迁移到 Monaco Editor 独立编辑器。
3. 编辑器实时解析或增量解析，展示错误、警告和大纲。
4. 保存后生成 IR、本地化提取表和热重载 Patch。
5. Unity 预览窗口接收 Patch，刷新可安全更新的指令堆栈。
6. 作者通过变量监视器和时间回放检查分支效果。

## 第一阶段界面候选

- 文件树：浏览脚本、角色、资源和本地化表。
- 编辑区：Monaco Editor，支持语法高亮、错误下划线、补全和悬浮说明。
- 大纲区：显示章节、节点、选择、跳转和角色出场。
- 诊断区：显示编译错误、资源缺失、未使用变量和本地化异常。
- 预览连接状态：显示 Unity 或未来运行时是否在线。

## 第二阶段界面候选

- 逻辑图：从文本结构生成可视化分支图，不替代文本源。
- 状态监视器：显示变量、标志位、当前锚点和执行历史。
- 时间回放：基于 Action 历史跳回任意剧本节点。
- 本地化面板：查看文本、译文、修改状态和疑似继承建议。
- 资源引用面板：检查背景、立绘、音频等引用是否存在。

## 交互待确认

- 编辑器的主交互是纯文本优先，还是文本 + 图双主视图。
- 逻辑图是否允许直接编辑，还是只读可视化。
- 实时解析是按键级、保存级，还是防抖后的增量解析。
- 热重载失败时的体验是什么：回滚、提示重启预览，还是局部禁用。
- 变量监视器是否允许直接编辑运行时变量。
- 时间回放如何处理随机数、异步资源加载和外部系统副作用。

## 技术待验证

- Tauri 后端监听文件变更与项目索引的性能。
- Monaco 自定义语言服务的诊断、补全和语义标记能力。
- WebSocket Patch 协议是否足够，是否需要 Socket.io 的重连和事件语义。
- Unity 预览端热重载最小协议。
- 大型剧本项目的索引、搜索和跨文件跳转性能。
