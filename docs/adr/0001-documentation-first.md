# 0001：采用文档先行的立项方式

状态：Accepted

## 背景

Inscape 同时包含 DSL、编译器、编辑器、运行时、本地化和未来独立引擎迁移能力。项目跨度较大，如果直接进入实现，容易把早期假设固化为难以修改的代码结构。

## 决定

项目先建立文档体系，再进入核心原型开发。文档体系包括项目立项、架构草案、DSL 草案、编辑器草案、运行时草案、本地化草案、路线图、待确认问题和 ADR。

## 原因

- 便于区分“已确定方向”和“仍需讨论的候选方案”。
- 便于后续围绕具体问题做设计收敛。
- 便于未来贡献者理解项目边界，而不是从代码中反推意图。
- 便于把语法、编辑器和运行时的关键取舍留下可追踪记录。

## 影响

- 早期仓库以文档为主，代码实现会在关键决策收敛后逐步加入。
- 影响持久化、存档、本地化和编辑器交互的改变需要新增 ADR。
- 文档需要持续维护，不能只作为一次性立项材料。
