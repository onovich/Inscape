# 0005：DSL 只表达状态查询，不绑定具体业务行为

状态：Accepted

## 背景

叙事分支常常需要查询玩家实体、物品、服务端状态或项目自定义系统。如果 DSL 直接内置变量和函数，容易把语言绑定到某个业务层或引擎实现。

## 决定

第一版不设计变量和条件查询。第二版开始设计时，DSL 只保存面向数据的查询表达式，由宿主层根据 Schema 解析并执行具体行为。

## 原因

- 保持 DSL 与 Unity、服务端和业务实体解耦。
- 避免控制反转，让上层系统保留自己的状态管理和查询方式。
- 让同一份 DSL 可以被 Unity、自研引擎或 HTML 调试预览以不同能力解释。

## 影响

- 查询表达式默认不允许副作用。
- 宿主层需要提供查询 Schema、诊断信息和运行时 resolver。
- HTML 预览可以先用 mock resolver 调试分支，不需要真实游戏状态。
