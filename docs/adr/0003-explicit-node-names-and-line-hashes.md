# 0003：块级使用显式节点名，行级使用隐式哈希

状态：Accepted

## 背景

Inscape 的 DSL 需要支持图叙事：节点可以重复进入，可以产生回环，也可以形成链、树或一般图。仅依赖隐式哈希可以让文本自动获得机器锚点，但作者很难稳定讨论、跳转和调试某个叙事块。

## 决定

块级叙事单元使用显式节点名。行级对白、旁白和选项文本使用隐式哈希锚点。

## 原因

- 显式节点名适合人类组织图结构、写跳转、讨论剧情块和查看大纲。
- 行级隐式哈希适合本地化、存档、热重载和文本修订追踪。
- 两者职责不同，不应该互相替代。

## 影响

- 第一版语法必须包含节点定义和节点引用。
- 编译器需要校验重复节点、缺失节点、不可达节点和循环关系。
- 源映射需要同时记录节点名和行级哈希。
- 后续仍需确认显式节点名是否生成独立机器 ID，以及节点重命名如何迁移存档。
