# 0002：把未定语法和交互显式标注为草案

状态：Accepted

## 背景

项目当前已经明确核心哲学与大致技术方向，但 DSL 语法和编辑器交互尚未完成设计。如果在文档中把示例语法和界面设想写成最终规范，会误导后续实现。

## 决定

所有尚未确认的语法、编辑器交互、热重载协议、哈希公式和运行时扩展机制，都必须标注为草案、候选或待确认。

## 原因

- 保留设计空间。
- 避免项目早期过度承诺。
- 让后续讨论聚焦在明确的问题上。
- 允许用原型和样例脚本验证语法，而不是只凭直觉定稿。

## 影响

- 文档中可以包含示例，但示例不自动成为规范。
- 实现前需要从待确认清单中挑选问题进行决策。
- 当某个设计从草案变为基线，应更新文档状态并补充 ADR。
