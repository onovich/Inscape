# 0004：DSL 阶段提供 VSCode 支持和 HTML 调试预览

状态：Accepted

## 背景

独立编辑器属于项目第三步，但 DSL 第一阶段如果只有纯文本和命令行，会很难快速验证阅读体验、节点跳转、回环和错误提示。用户预期在 VSCode 中编辑文本，并通过高亮让次要信息变弱、剧情文本更突出。

## 决定

DSL 阶段提供 VSCode 语言支持和轻量 HTML 调试预览。它们是语言设计和调试工具，不等同于最终独立编辑器。

## 原因

- 语法高亮和代码提示能降低记忆压力。
- HTML 预览可以在无 Unity 环境下验证叙事图。
- 早期反馈能帮助语法收敛，避免等到编辑器阶段才发现语言不好写。

## 影响

- Compiler Core 需要适合被 Language Server 和预览服务调用。
- IR 需要足够稳定，让 HTML 预览直接消费。
- 第一阶段工作范围包含 TextMate 高亮、基础 LSP 能力和预览原型。
